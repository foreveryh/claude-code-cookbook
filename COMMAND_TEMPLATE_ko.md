## 명령어 이름

명령어가 수행하는 작업에 대한 간결한 설명 (한 줄 설명).

### 사용법

```bash
/command-name [옵션]
```

### 옵션

- 없음 : 기본 동작
- `--option1` : 옵션 1 설명
- `--option2 <값>` : 매개변수가 있는 옵션 설명
- `--flag` : 플래그 옵션 설명

*참고: 명령어에 옵션이 없는 경우 이 섹션을 생략할 수 있습니다.*

### 기본 예제

```bash
# 기본 사용법
/command-name
"실행 시 설명"

# 옵션 사용 예제
/command-name --option1
"옵션 1 사용 시 설명"

# 매개변수가 있는 옵션 예제
/command-name --option2 value
"매개변수가 있는 옵션 사용 예제"

# 여러 옵션 조합
/command-name --option1 --flag
"여러 옵션을 조합한 예제"
```

### 고급 기능 (선택사항)

#### 기능 카테고리 1

기능에 대한 자세한 설명과 사용 사례.

```bash
# 구체적인 코드 예제
command example
```

#### 기능 카테고리 2

다른 기능 설명.

- **특징 1**: 설명
- **특징 2**: 설명
- **특징 3**: 설명

*참고: 단순한 명령어의 경우 이 섹션을 생략할 수 있습니다.*

### 출력 예제 (선택사항)

```
예상 출력 형식 예제
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

실제 출력 샘플
├─ 항목 1: 값
├─ 항목 2: 값
└─ 항목 3: 값
```

*참고: 출력이 단순하거나 기본 예제에서 충분히 설명된 경우 이 섹션을 생략할 수 있습니다.*

### Claude 통합

```bash
# 파일 내용과 함께 사용
cat file.txt
/command-name
"파일 내용을 포함한 분석 요청"

# 파이프라인에서 사용
command1 | command2
/command-name
"이전 명령어의 출력을 사용한 처리"

# 복잡한 분석 예제
ls -la && cat config.json
/command-name --option1
"여러 정보 소스를 결합한 분석"
```

### 고려사항

- **사전 요구사항**: 명령어 실행에 필요한 요구사항
- **제한사항**: 기능 제한이나 주의할 점
- **권장사항**: 효과적인 사용을 위한 권장사항

### 모범 사례 (선택사항)

1. **원칙 1**: 권장 사용 패턴 설명
2. **원칙 2**: 피해야 할 패턴 설명
3. **원칙 3**: 성능 및 효율성 권장사항
4. **원칙 4**: 보안 및 안전 권장사항

*참고: 특별한 고려사항이 없는 단순한 명령어의 경우 이 섹션을 생략할 수 있습니다.*

### 관련 명령어 (선택사항)

- `/related-command1` : 관련 기능을 가진 명령어
- `/related-command2` : 함께 사용하면 좋은 명령어
- `/related-command3` : 대체 가능한 명령어

*참고: 관련 명령어가 없는 경우 이 섹션을 생략할 수 있습니다.*
